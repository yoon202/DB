# 학생 출결 관리 데이터베이스 (Student Attendance Management DB)

## 1. 프로젝트 개요
본 프로젝트는 대학 강의 환경에서 학생 출결을 체계적으로 관리하기 위한  
**학생 출결 관리 데이터베이스의 설계 및 구현**을 목표로 한다.

학생, 교수, 강의, 수강, 출결 정보를 관계형 데이터베이스로 구성하여  
출결 등록, 조회 및 통계 분석이 가능하도록 설계하였다.

본 프로젝트는 데이터베이스 설계 과목의 기말 과제로 수행되었으며,  
ERD 설계부터 정규화, 물리적 설계, SQL 구현까지의 전 과정을 포함한다.

---

## 2. 개발 환경
- **DBMS** : Oracle Database
- **개발 도구** : Oracle SQL Developer
- **사용 언어** : SQL (DDL / DML)
- **운영체제** : Windows

---

## 3. 데이터베이스 구성

### 3.1 테이블 목록 및 역할

| 테이블명 | 설명 |
|----------|------|
| STUDENT | 학생 기본 정보 관리 |
| PROFESSOR | 교수 정보 관리 |
| COURSE | 강의(과목) 정보 관리 |
| ENROLL | 학생 수강 신청 정보 관리 |
| ATTENDANCE | 수업 일자별 출결 정보 관리 |

---

### 3.2 테이블 간 관계

- STUDENT 1 : N ENROLL  
- COURSE 1 : N ENROLL  
- COURSE 1 : 1 PROFESSOR  
- ENROLL 1 : N ATTENDANCE  

본 데이터베이스는 Crow’s-feet 표기법을 기반으로 ERD를 설계하였으며,  
모든 테이블은 외래키 제약조건을 통해 참조 무결성을 유지한다.

---

## 4. 설계 특징

### 4.1 정규화
- 비정규형 데이터 구조를 분석하여 제1정규형(1NF), 제2정규형(2NF)을 거쳐
- **제3정규형(3NF)** 까지 정규화를 수행하였다.
- 데이터 중복을 최소화하고 삽입·수정·삭제 이상이 발생하지 않도록 설계하였다.

### 4.2 물리적 설계
- 기본키에 대해 DBMS 자동 인덱스를 활용하였다.
- 조회 및 조인 빈도가 높은 컬럼에 대해 보조 인덱스를 추가로 정의하였다.
- 데이터 정합성과 유지보수를 고려하여 **역정규화는 적용하지 않았다.**

---

## 5. 주요 기능 (SQL 기반)

### 5.1 데이터 관리
- 학생, 교수, 강의 정보 등록 및 관리
- 학생의 수강 신청 정보 관리
- 수업 일자별 출결 상태 관리 (출석 / 지각 / 결석)

### 5.2 데이터 조회
- 학생별 출결 현황 조회
- 강의별 출결 현황 조회
- 교수별 담당 강의 및 출결 조회
- 학생 및 강의별 출석률 계산

### 5.3 통계 분석
- 학생별 출석률 통계
- 강의별 평균 출석률
- 결석 및 지각 발생 학생 조회

## 6. SQL 스크립트 구성

| 파일명 | 설명 |
|------|------|
| `attendance_ddl.sql` | 테이블, 제약조건, 시퀀스, 인덱스 생성 스크립트 |
| `attendance_data.sql` | 테스트용 초기 데이터 입력 스크립트 |
| `attendance_queries.sql` | SELECT / UPDATE / DELETE SQL 문 (30문제 이상) |

모든 SQL 스크립트는 **전체 실행만으로 데이터베이스가 정상적으로 구축**되도록 작성되었다.
